Governance

Governance defines who may issue identifiers, how records change, and how stability is preserved. The registry is append-only: identity is permanent; updates are recorded as new facts and new audit entries.

Roles

  • Authority: E000001
  • Registrar: A000001
  • Operator: optional (infrastructure only, no semantic power)
  • Reviewer: optional (quality gate, does not issue IDs)

Change control

  • Never delete or reuse identifiers.
  • Every issuance adds: /data, /page, /id, and an audit entry.
  • Edits to human pages are allowed, but must not change identifier meaning.
  • Schema/ontology changes are additive; v1 core remains locked.

Audit

  • Audit entries live in /audit/logs/ as sequential JSON files.
  • audit/logs/index.json points to the latest entry.
