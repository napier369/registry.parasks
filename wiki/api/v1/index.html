API v1 — Stability Contract

API v1 is frozen. It will never change in meaning, hash rules, or endpoint semantics. New capabilities happen in v2+ only.

Frozen guarantees

  • Identifier invariance: /id/{ID}/ remains permanent.
  • Resolver contract: /id/{ID}/ keeps resolving to /page and /data.
  • Canonical hash rule (v1): SHA-256 of JSON-LD using sorted keys, minimal separators, UTF-8, with contentHash set to sha256:REPLACED_AT_RUNTIME.
  • Bundle truth: /api/v1/bundle/{ID}.json carries the sealed contentHash for verification.
  • Manifest truth: /api/v1/manifest.json lists sealed hashes for all IDs.

Non-goals (v1 will not grow)

  • No new negotiation modes (RDF accept headers etc).
  • No schema/field renames.
  • No hash-rule revisions.
