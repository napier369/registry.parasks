Public API

The registry is published as static, read-only resources. There is no write API. Publication occurs by issuing new records and audit entries.

Endpoints

  • /id/<ID>/ — identifier entrypoint (resolves to a human page)
  • /page/<ID>/ — human-readable page
  • /data/<ID>.jsonld — machine-readable JSON-LD record
  • /audit/logs/index.json — pointer to latest audit entry
  • /audit/logs/00000N.json — audit entries
  • /.well-known/registry-keys.json — verification keys (present/future)

Usage

  • Humans: browse /id or /page
  • Machines: fetch /data JSON-LD
  • Monitoring: poll /audit/logs/index.json for changes

Status codes

  • 200 — resource exists
  • 404 — resource not present
