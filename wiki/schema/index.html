Schema & ontology

This registry uses a minimal core ontology for consistent machine-readable records. The ontology is published under /ontology/v1/ and records are published as JSON-LD under /data/.

Core classes

  • Entity (E) — things that exist (authority, policy, etc.)
  • Agent (A) — actors (registrar, operator)
  • Document (D) — governed texts and instruments
  • Event (X) — actions recorded in time (issue, genesis)

Core fields (minimum)

  • @context — ontology/v1/context.jsonld
  • @id — canonical identifier URL (/id/<ID>)
  • @type — Entity/Agent/Document/Event
  • rdfs:label — human name
  • registry:issuedAt — issuance timestamp
  • registry:status — active/deprecated/historical

Where truth lives

  • /id — identity (permanent entrypoint)
  • /data — machine truth (authoritative JSON-LD)
  • /page — human view (readable rendering)
  • /audit — append-only change record
