Monitoring

Monitoring is based on the append-only audit log. Watch the latest pointer, then fetch the referenced audit entry.

Step-by-step

  1. Fetch: /audit/logs/index.json
  2. Read latestSeq and latest
  3. Fetch the latest file (e.g. audit/logs/000003.json)
  4. Process the adds list to discover new IDs
  5. For each new ID, fetch /data/<ID>.jsonld

Notes

  • Audit is append-only; old entries are never edited.
  • If the pointer moves forward, something new was published.
  • Project hosting is static; availability depends on the hosting platform.
