Contributing

Contributions are additive only. Do not rename, delete, or reuse identifiers. New records are issued by publishing a complete record pack and an audit entry.

Record pack (required)

  • data/<ID>.jsonld — authoritative machine record
  • page/<ID>/index.html — human-readable view
  • id/<ID>/index.html — identifier resolver stub
  • audit/logs/<SEQ>.json — append-only audit entry

Rules

  • IDs are opaque and sequential (no meaning encoded).
  • Every new ID must be added to an audit entry.
  • Update audit/logs/index.json to point to the latest audit entry.
  • Update wiki/index.html to link new documentation pages (optional).

Minimum JSON-LD fields

  • @context
  • @id
  • @type
  • rdfs:label
  • registry:issuedAt
  • registry:status
