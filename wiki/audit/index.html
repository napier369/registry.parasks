Audit & Verification

Every issuance and seal action is append-only and recorded in /audit/logs/. The current audit head is /audit/logs/index.json.

What to fetch

  • Audit head: /audit/logs/index.json
  • Latest audit entry: value of latest inside the audit head
  • Canonical machine record: /data/<ID>.jsonld

Verification rule

  • Each /data/<ID>.jsonld contains contentHash.
  • Verification passes when the SHA-256 of the canonical JSON (with contentHash blanked to sha256:) equals the stored hash.

Live examples

  • E000003: data | id | page
  • E000004: data | id | page

Tools

  • Local validator: python3 tools/issue_check.py <ID>
  • Latest audit reader: python3 tools/reg_get_latest.py
