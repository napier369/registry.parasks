How it works

IDENTITY (permanent)
  /id/E000001/
      |
      | (resolver)
      v
HUMAN VIEW            MACHINE TRUTH
/page/E000001/        /data/E000001.jsonld
  • /id never changes.
  • /page can change layout without changing identity.
  • /data is the authoritative machine form.

Publishing surfaces

  • /id/<ID>/ → redirects to /page/<ID>/
  • /page/<ID>/ = human-readable page for the ID
  • /data/<ID>.jsonld = canonical machine record (JSON-LD)
  • /audit/logs/index.json = audit head pointer
  • /audit/logs/00000N.json = append-only audit events
  • /api/v1/index.json = discovery index of published records
  • /wiki/ = narrative documentation (non-authoritative)

Fetch flow (example)

  • Start: /id/E000004/
  • Machine: /data/E000004.jsonld
  • Audit head: /audit/logs/index.json
  • Discovery: /api/v1/index.json

Issuance rule

  • Issuing a new ID adds: /data, /page, /id, and an audit entry.
  • Identifiers are never deleted or reused.
  • Wiki edits must not change identifier meaning.
